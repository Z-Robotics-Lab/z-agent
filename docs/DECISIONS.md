## D3 — Hardware Abstraction Layer · Accepted 2026-03-25 (live from v2.0)
- Hardware-agnostic: SO-101 and Go2 are interchangeable adapters into one Agent/Skill/LLM pipeline.
- Formalized `BaseProtocol` + `PerceptionProtocol` (alongside `ArmProtocol`/`GripperProtocol`); dict-based `SkillContext`.
- Rejected ad-hoc `base: Any` duck-typing — brittle as hardware variants multiply, blocks capability queries.

## D4 — Web Visualization replaces RViz · Accepted (revised) 2026-04-09
- ROS2 web viz via **Foxglove Studio** (revised away from a self-built Three.js viewer).

## D5 — Isaac Sim Integration · Superseded / Paused
- Docker'd Isaac as a high-fidelity sim backend — **paused/archived** during the v2.4 perception pivot.
- **MuJoCo remains the primary sim** for dev + CI; the in-tree Isaac proxy is an interface/topic contract only (no working Isaac SDK code).

## D6 — Agent Kernel / World-Plugin · Accepted 2026-06-05
- Engine + VGG + tools + backends + session + permissions = a **model-/skill-/hardware-agnostic orchestration KERNEL**; each deployment target = a **world plugin**.
- A world registers exactly 4 things: tools, verify/primitive namespace, decompose vocabulary, persona. **The kernel never imports a world.**
- A default `dev`/`code` world ships in-kernel (a build/test means, not the product); the robot is an optional world plugin. Migration, not rewrite.

## D7 — Open-Loop Compiler → Grounded Closed-Loop Controller · Accepted 2026-06-06 (staged)
- Re-architect VGG from a blind one-shot compiler into a grounded closed-loop controller via 5 bets:
  - **Observation loop:** per-run `Blackboard`; `GoalVerifier` returns `(bool, raw_value)`; `StepRecord.result_data`; `${step.output.path}` binding (no f-strings over data).
  - **Living plan:** control-flow IR (`foreach`/`until`/`if`) + observation-driven mid-tree replan from current `world_context`, not the stale T=0 context.
  - **Single-source action space** from the skill registry (kill the Go2 split-brain vocab + keyword sieves); fail-loud `validation_notes` fed back into replan.
  - **Ground the planner:** typed perception facts, referring-expression resolution, coherent `ObjectMemory`.
  - **Unify planning paths:** merge `VectorEngine` + `VGGHarness` into one controller.
- NOTE: D9 supersedes the bespoke-planner direction — the MODEL, not hand-written decompose/replan code, now owns plan/route/recover (D7's closed-loop *grounding/observation* ideas still hold; its hand-authored planner does not).

## D8 — Playground as a Parallel Track via Seam-as-Contract · Accepted 2026-06-06
- Playground developed as a **separate parallel track**; the kernel/world **seam is the versioned public contract** the tracks integrate across.
- Extend the seam with a **verified-loop observation surface** (GoalTree + per-step StepRecord + replan notes for a front-end to render).
- v1 = **preset scenes** entered via NL (agent owns the long chain once inside; dynamic scene assembly deferred); deterministic sim-oracle verify + VLM perception (generator/verifier independence).
- NOTE: `feat/playground-vln` was later **abandoned** (never merged); `master` is the base (2026-06-18 reframe).

## D9 — Native Model-Driven Tool-Use Producer IS the Orchestration Design · Accepted (CEO ruling) 2026-06-19
- **The native model-driven tool-use producer is the correct design; the legacy hardcoded planner (`should_use_vgg`→`vgg_decompose`→direct skill execution) is WRONG and is being strangled.** Going forward: **use the native producer and optimize it iteratively — never retreat to the legacy planner for any capability.** (CEO ruling.)
- Why: routing/decompose/recover belong to the MODEL (the North Star), not bespoke keyword code; re-implementing planning in code is the drift D7 + the 2026-06-18 reframe identified.
- **Cutover (done in the REPL):** bare `vector-cli` + NL runs native by default — `cli.run_turn_unified` attempts native-first, falls back to legacy ONLY on a zero-action turn; `VECTOR_REPL_NATIVE=0` forces pure-legacy. Legacy VGG stays only as the strangler-fig fallback, pending staged reversible deletion. Merge-to-master = separate CEO gate.
- Mechanism: `vcli/native_loop.py run_turn_native` — the MODEL drives a ReAct loop (world skills + registry code tools + synthetic `verify`/`finish`); assembles an `ExecutionTrace` the honest verify spine grades BYTE-UNCHANGED. Never computes `verified` itself (moat only ever STRICTER, rule 5).
- **Known native improvements (optimize — NOT reasons to retreat to legacy):**
  - **Nav avoidance:** native locomotion uses the open-loop `walk_forward` (no lidar/planner) → `走到坐标 (x,y)` walks straight into obstacles (e.g. `pick_table` @ (11,3), go2 starts (10,3)). NEXT = route native go-to-place through the nav-stack avoidance route (`publish_goal`→FAR/local-planner/lidar); cmd_vel motion grades RAN until actor-causation extends to cmd_vel.
  - **Latency:** native = several LLM round-trips + currently SYNCHRONOUS (blocks the REPL) vs legacy async/responsive → optimize to async + fewer round-trips.

## D10 — Recover-by-retry is FAIL-CLOSED by design (the honest characterization) · Accepted 2026-06-20
- The native producer's RECOVER pillar (north-star #4) is honest BY CONSTRUCTION, proven on real go2 sim via `cli.main` PTY (`tests/vcli/test_native_loop_recover_pty.py`): a fake "done" (finish while the latest verify FAILs) grades RAN/verified False; a never-landing recovery TERMINATES (backend-exhaust `end_turn`, `max_turns=24` never hit) all-RAN; a GENUINE in-turn recovery (walk-short→verify FAIL→walk-again→verify PASS→finish) records `[RAN, GROUNDED]` and grades **verified False** — the all-GROUNDED gate (`trace_store.evidence_passed`) fails CLOSED on the recorded FAIL row (`native_loop` records one StepRecord per verify, no dedup; a verify closes the step and resets the baseline).
- CONSEQUENCE (made explicit, not hidden): a deterministic recover-to-verified-True is UNREACHABLE without a per-sub-goal latest-verify gate that lets a later PASS overwrite a recorded FAIL — that is LOOSER, and rule 5 (verify only ever stricter) forbids it. The spine stays BYTE-UNCHANGED. Recover-to-verified-True is demonstrated ONLY via the LIVE path where the model verifies once after recovering (`test_native_loop_multistep_live_pty`). The moat never falsely greens a recovery; it can only be stricter than the model.
- Goal-authenticity of the verify constant (does the (x,y) match the real task goal) remains the deferred residual — unchanged by this decision.

## D11 — Two moat false-greens closed stricter-only (2nd adversarial review) · Accepted 2026-06-20
- The STEP-12 2nd adversarial moat review (Opus skeptic panel) found TWO genuine, independent false-greens that graded a FALSE turn `verified=True`; both fixed STRICTER-only (rule 5), honest cases preserved, each proven flipped to RAN/exit 2 through the REAL `cli.main` on go2 sim:
  - **(A) Compare-membership short-circuit** (`evidence_classifier._is_grounded_node`): the STEP-8 every-leaf-oracle-gated recursion descended BoolOp/UnaryOp but NOT the `Compare` branch, which stayed a flat `has_oracle and has_const and not oracle_only`. So `True in (at_position(99,99), True)` (and `<oracle> in <constant-literal container>` siblings) classed GROUNDED — the constant member satisfies membership, the oracle is dead weight (the `... or True` short-circuit hidden in an `in` node). FIX: membership (`in`/`not in`) is grounded ONLY if the CONTAINER is oracle-derived (`'table' in describe_scene()`); a constant-literal container (tuple/list/set/dict) → RAN.
  - **(B) facing/visited cross-channel causation** (`actor_causation._base_displacement`): graded EVERY base predicate on `max(dxy, dyaw)`, so a forward walk (dxy>0, dyaw~0) laundered a satisfied-at-baseline `facing(h)` no-op to CAUSED — defeating R2b's own purpose for the heading channel. FIX: per-channel — `facing` keys on heading (`dyaw >= _YAW_CAUSE_EPS` ~10°, above gait drift / below an intentional turn), `at_position`/`visited` key on planar (`dxy >= DISPLACEMENT_EPS`). Bonus: a pure turn no longer false-greens `at_position`.
- REMAINING (deferred, documented, NOT closed): (i) a MINOR `visited(room)` intra-room-walk residual — an already-in-room walk has dxy>0 so still grades CAUSED; the honest fix needs evaluating the verify oracle at BASELINE to require a False→True membership transition (a deeper grade/capture change). (ii) The goal-authenticity residual (a real CAUSED action + a trivially-true verify over the actor's OWN resulting state, e.g. `at_position(<landing>)` / `facing(<current yaw>)` / `len(get_position())==3`) is UNCHANGED — it needs the parsed real task goal as structured data, a semantic axis the purely-structural+causation moat cannot check.
- The other review lenses were DRY: ternary/walrus/any-all/comprehension/subscript short-circuits are fail-closed by the unhandled-node default; the fallback/verdict path single-grades a turn and delegates `verified` verbatim to `evidence_passed`; delivered capabilities (walk/pick/cross-lang/recover) hold against no-op/teleport/wrong-object.

## D12 — Goal-authenticity for coordinate goals (the verify constant must match the user's target) · Accepted 2026-06-20
- STEP-13 closes the deepest repeatedly-deferred moat residual for COORDINATE goals: the grader downgrades an otherwise-GROUNDED `at_position` step to RAN when the verify CONSTANT does not match the user's PARSED coordinate goal (within the oracle tolerance). So a model can no longer do a real actor-caused walk to the WRONG place and self-certify `at_position(<its own landing>)`. INTRA-PACKAGE (vcli/cognitive), STRICTER-only (rule 5), NO CEO gate — the NL goal (`goal_tree.goal` = user_message verbatim, native_loop.py:357) already co-travels with the verify expr in one ExecutionTrace; the check lives in `classify_step_evidence` (trace_store.py) mirroring the R2b downgrade, respecting native_loop's import firewall.
- Mechanism: new pure `vcli/cognitive/coord_goal.py` — `parse_goal_coord` (regex first `(x,y)` in zh/en; None on absent/ambiguous), `at_position_const` (ast; literal `at_position(x,y)` only), `coord_goal_mismatch` (both parse AND `dist > tol`; tol single-sourced by re-reading `_AT_POSITION_TOL_M`). `classify_step_evidence` gains a `goal_text` kw-default (rule 6 additive), threaded from `evidence_passed` + `VerdictReport.from_trace`; the per-step reward gate passes None. FAIL-OPEN whenever either side is not a parseable coordinate, so dev / grasp / facing / coordinate-less turns are untouched.
- Honest by construction: the target is parsed from the USER's command words, never actor-authored. Red-team: (a) walk to L + verify `at_position(L)` for goal (11,3) → mismatch → RAN; (b) verify `at_position(11,3)` while standing at L → const==goal so no mismatch, BUT verify_result is the live oracle (`dist(L,(11,3))>tol` → False) → RAN via R1. The only GROUNDED path is to actually be at the commanded coordinate. Proven on real go2 sim via cli.main (wrong-place→RAN/exit2; honest→GROUNDED; fail-open facing-under-coord-goal→GROUNDED).
- REMAINING (deferred, documented, NOT closed): (i) WRONG-PREDICATE-TYPE — for a coordinate goal a model could verify a DIFFERENT predicate (`facing(<current>)`, `len(get_position())==3`) and pass via R1+R2b without proving it reached the coordinate; closing it needs per-sub-goal goal awareness (the turn goal alone cannot tell which step should be `at_position`, cf. the legit multiskill facing step). (ii) non-coordinate goals (no parenthesized coords) and object/room goal authenticity (`holding_object`/`visited` target matching).

## D13 — Callable-container / reduction short-circuit closed (3rd moat review) · Accepted 2026-06-20
- The STEP-14 3rd adversarial review found the STEP-12 membership/short-circuit fix was INCOMPLETE: it rejected only LITERAL `ast.Tuple/List/Set/Dict` containers, so a builtin container constructor (an `ast.Call`) bypassed it. Two CRITICAL false-greens, both GROUNDED + sandbox-True while the robot was nowhere near the goal:
  - `True in tuple((at_position(X,Y), True))` — the constant `True` member satisfies the `in`; the oracle is dead weight (the `... or True` short-circuit wrapped in a `tuple()` call).
  - `len(tuple((at_position(X,Y), True))) == 2` — `len()` over a constant-structure tuple is always 2; the oracle's value is discarded.
- FIX (stricter-only, rule 5): a new `_truth_bearing_oracle` predicate — an oracle counts toward GROUNDED only when its VALUE reaches the operand: a direct oracle call, an index / arithmetic over one (`get_position()[0]`), or a reduction whose ARGUMENT is directly truth-bearing (`len(detect_objects()) > 0` stays honest). An oracle buried inside a literal or builtin-constructed CONTAINER (`tuple((x, True))`) is inert — descend into call ARGS, never into literal container ELEMENTS. The membership branch additionally requires the container to be a DIRECT oracle call. Proven RAN through real cli.main on go2 sim + unit regressions; honest verifies (detect_objects/get_position()[0]/placed_count/get_heading<C) preserved. The shape-trivial `len(get_position())==3` is STRUCTURALLY indistinguishable from the honest `len(detect_objects())>0`, so it remains the documented xfail residual (deferred to a goal-aware R2).
- LESSON: fixing one member of a short-circuit family (literal containers) does NOT close the family — a builtin wrapper re-opens it. The robust fix is a semantic "does the oracle's value reach the truth" predicate, not an enumerated blocklist of node shapes.
- REMAINING (found this review, NOT fixed): (i) MAJOR wrong-predicate-type — a coordinate goal verified with a NON-`at_position` predicate (`facing(<current>)`, `len(get_position())==3`) grades GROUNDED on the DEFAULT native path without reaching the target; closing it needs per-sub-goal goal awareness (a turn-level "a coordinate goal requires a matching `at_position` GROUNDED step"). (ii) MAJOR legacy-producer goal-poisoning — on the legacy decompose path the model authors `goal_tree.goal`, so STEP-13 can be fed a self-matching goal; off the bare-cli acceptance default (legacy being strangled); fix = thread the verbatim user task as goal_text. (iii) MINOR multi-coordinate "from-to" phrasing makes STEP-13 fail open. The moat is NOT yet dry.

## D15 — Wrong-predicate-type closed: a coordinate goal must be VERIFIED with a matching at_position (turn-level gate) · Accepted 2026-06-20
- STEP-15 closes the MAJOR wrong-predicate-type residual flagged by D13: on the default native path a COORDINATE goal (`走到坐标 (11,3)`) could be verified ENTIRELY with a NON-`at_position` predicate (`facing(<current>)` after a real CAUSED turn, or `len(get_position())==3`) and pass — every step graded GROUNDED per-step (the action is honestly actor-caused), and STEP-13's per-step coordinate mismatch downgrade FAILS OPEN on a non-`at_position` verify, so the commanded coordinate is never asserted. A TURN-level gate is required because the per-step grader cannot tell which step *should* be the `at_position` (a legit multiskill turn-then-face has an honest `facing` step alongside its `at_position` leg).
- FIX (stricter-only, rule 5, intra-package vcli/cognitive): `evidence_passed` gains a coordinate turn-gate that runs ONLY after the all-GROUNDED rule already holds (so it can only REJECT, never fake-pass). SCOPED to a go2-coordinate world (`at_position` in the live oracle set, rule 3) so dev/arm/answer worlds with an incidental `(x,y)` in the goal text are world-blind/untouched. When `parse_goal_coord` pins the target, require ≥1 checked step that is BOTH GROUNDED and whose verify is a bounded-tol literal `at_position` matching the commanded coord (new `coord_goal.at_position_const_matches`). When the goal is coordinate-shaped but the regex cannot pin a single target (paren-less `导航到 11,3`, or ≥2 distinct coords — new `goal_has_coordinate_intent`), still require SOME GROUNDED literal `at_position` step (the facing-only escape is closed even if the exact coord can't be pinned). A coordinate-less go2 goal (room name / explore / turn) fails OPEN, unchanged.
- Bonus hardening to `at_position_const` (same stricter-only intent): (i) KWARG form `at_position(x=11,y=3)` is now read (previously extracted None → fail open → a wrong-coord kwarg verify passed); (ii) TOL-INFLATION `at_position(11,3,99)` / `tol=99` (an explicit arrival tolerance literal above `_MAX_ARRIVAL_TOL_M`=2.0 m makes the predicate "within 99 m" ≈ true anywhere) returns None → fail CLOSED for every downstream coordinate check; a non-literal (state-derived) tol is likewise rejected. Generous bound (accepts honest 0.5–2 m), rejects only an EGREGIOUS tolerance.
- The gate cannot be gamed by a dummy `at_position` step the robot didn't reach: an UNCAUSED (teleport/no-op) `at_position` is already downgraded to RAN per-step, so it fails the all-GROUNDED rule before the coordinate clause is reached.
- Proven on real go2 sim via `cli.main` PTY: a coordinate goal verified ONLY with `facing()` after a real left turn → the facing step stays honestly GROUNDED, yet the TURN is verified False / RAN / exit 2 (the turn-level gate rejects a coordinate turn that never asserts the coordinate). 53 coord_goal unit + 708 unit/vcli + 601 non-sim vcli green (the 3 deepseek + 1 level71 reds are the documented pre-existing environmental reds, not regressions).
- REMAINING (deferred, NOT closed): (i) shape-trivial `len(get_position())==3` stays the documented xfail (structurally == honest `len(detect_objects())>0`); under a coordinate goal the turn-gate now rejects it at turn level, but the per-step GROUNDED classification of the lone trivial verify is unchanged. (ii) legacy-producer goal-poisoning (off the bare-cli acceptance default; legacy being strangled). (iii) object/room (non-coordinate) goal authenticity. The next moat review (4th) judges whether the short-circuit/goal-authenticity family is finally dry.

## D16 — Boolean-necessity closed: a coordinate goal's matching at_position must be a NECESSARY conjunct, not merely PRESENT (4th moat review) · Accepted 2026-06-20
- The 4th adversarial moat review found the STEP-13/15 coordinate gates checked only that a goal-matching `at_position` constant was PRESENT in the verify (`at_position_const(...) is not None`, the first `ast.walk` hit), NOT that it was NECESSARY for the verify's truth. So a real CAUSED walk to the WRONG place + a verify whose goal-matching `at_position(11,3)` is buried under a disjunction / negation / arithmetic threshold graded verified=True while the robot stood elsewhere. Independently reproduced 10 variants: `at_position(11,3) or visited(99,99)`, `... or at_position(99,99)`, `not at_position(11,3) or ...`, `(at_position(11,3) or visited(0,0)) and facing(1,0)`, `at_position(11,3) + at_position(99,99) >= 1`, etc. — all PRESENT-but-inert, all formerly passing the gate.
- FIX (stricter-only, rule 5, intra-package vcli/cognitive): new `_necessary_at_position_consts` — the (x,y) consts of every `at_position` reachable from the verify root through `BoolOp(And)` nodes ONLY. Such a call's falsity forces the whole verify False (an and-chain is True only if every conjunct is True), so it genuinely GATES the verdict on reaching the coordinate; an `at_position` under an `or` (a satisfiable sibling makes it non-necessary), a `not`, an arithmetic BinOp/Compare threshold, or an IfExp is excluded. `at_position_is_necessary(expr, goal_xy, tol)` / `has_necessary_at_position(expr)` replace the mere-presence checks: `at_position_const_matches` (turn-gate POSITIVE dual) and `coord_goal_mismatch` (per-step downgrade) now demand NECESSITY; `evidence_passed`'s parse-evasion branch uses `has_necessary_at_position` instead of `at_position_const is not None`. `at_position_const` stays the single-constant accessor only where presence is what's wanted. Purely structural.
- Honest by construction: necessity is a property of the verify's boolean structure (model-authored) measured against the user-parsed goal coordinate — no oracle is trusted. Red-team: the OR-decoy `at_position(11,3) or at_position(0,0,1000000)` EVALUATES True (huge-tol disjunct always true) after a real CAUSED forward walk, so R1 (verify True) + R2b (CAUSED planar walk) both pass and the (11,3) constant is PRESENT — yet the necessity gate rejects it because the matching constant is not an AND-conjunct. Only actually being at the commanded coordinate under a gating `at_position` survives.
- Proven on real go2 sim via `cli.main` PTY (test (F) `test_or_decoy_non_necessary_at_position_flips_to_ran`, 360s): the OR-decoy → RAN / verified False / exit 2, downgraded at both the per-step (`coord_goal_mismatch`) and turn-gate levels. 78 coord_goal unit (10 decoy + honest-conjunction cases) + 1335 non-sim vcli/unit green (only the 3 documented pre-existing deepseek env reds); honest conjunctions (`at_position(g) and facing(...)`, nested-OR-sibling, kwarg form) stay verified.
- LANDING NOTE: this fix was written by a prior loop round that crashed (API rate limit) before committing; this round RE-VERIFIED it from scratch (independent in-process probes + full unit suite + the real-sim PTY anchor) and LANDED it — per the recurring resume-invariant lesson (don't trust a dead round's claim; re-verify then land).
- LESSON: PRESENT ≠ NECESSARY. A goal-matching constant that does not GATE the verdict is a decoy; the honest test is boolean necessity (reachable through AND only), not lexical presence. This is the same "fix the family, not one member" pattern as D13 (callable-container) — a 4th consecutive review finding a structural false-green, so the moat is **still NOT dry**.
- REMAINING (deferred, NOT closed): (i) shape-trivial `len(get_position())==3` xfail (unchanged). (ii) multi-coordinate "from-to" goals where `parse_goal_coord` cannot pin a single target fall back to `has_necessary_at_position` (any necessary literal at_position) — the wrong-VALUE multi-coord case is a parse-asymmetry residual. (iii) legacy-producer goal-poisoning; (iv) object/room (non-coordinate) goal authenticity. A 5th review is needed to judge dry.
