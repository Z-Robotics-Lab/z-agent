## D6 — Agent Kernel / World-Plugin · Accepted 2026-06-05
- Engine + VGG + tools + backends + session + permissions = a **model-/skill-/hardware-agnostic orchestration KERNEL**; each deployment target = a **world plugin**.
- A world registers exactly 4 things: tools, verify/primitive namespace, decompose vocabulary, persona. **The kernel never imports a world.** A default `dev`/`code` world ships in-kernel (a means, not the product); the robot is an optional world plugin. Migration, not rewrite.

- NOTE: **D9 supersedes the bespoke-planner direction** — the MODEL, not hand-written decompose/replan code, owns plan/route/recover (D7's closed-loop grounding/observation ideas still hold; its hand-authored planner does not).

## D9 — Native Model-Driven Tool-Use Producer IS the Orchestration Design · Accepted (CEO ruling) 2026-06-19

## D10 — Recover-by-retry is FAIL-CLOSED by design · Accepted 2026-06-20
- The RECOVER pillar is honest BY CONSTRUCTION (proven on real go2 sim via PTY): a fake "done" (finish while latest verify FAILs) grades RAN/verified False; a never-landing recovery TERMINATES (backend-exhaust, max_turns never hit); a GENUINE in-turn recovery records `[RAN, GROUNDED]` and grades **verified False** — the all-GROUNDED gate fails CLOSED on the recorded FAIL row.
- CONSEQUENCE: a deterministic recover-to-verified-True is UNREACHABLE without a per-sub-goal latest-verify gate (LOOSER, forbidden by rule 5). Recover-to-verified-True is shown only on the LIVE path where the model verifies once after recovering. Goal-authenticity of the verify constant stays the deferred residual.

## D11 — Two moat false-greens closed stricter-only (2nd adversarial review) · Accepted 2026-06-20
- 2nd adversarial moat review found TWO independent false-greens grading a FALSE turn `verified=True`; both fixed STRICTER-only (rule 5), proven flipped to RAN on real go2 sim:
  - (A) **Compare-membership short-circuit**: `True in (oracle, True)` classed GROUNDED via the constant member. FIX: membership (`in`/`not in`) is grounded only if the CONTAINER is oracle-derived; a constant-literal container → RAN.
  - (B) **facing/visited cross-channel causation**: a forward walk laundered a satisfied-at-baseline `facing` no-op to CAUSED. FIX: per-channel — `facing` keys on heading (dyaw), `at_position`/`visited` on planar (dxy).
- REMAINING (open): a minor `visited(room)` intra-room residual; the goal-authenticity residual (a trivially-true verify over the actor's own resulting state) — needs the parsed task goal as structured data.

## D12 — Goal-authenticity for coordinate goals · Accepted 2026-06-20
- Downgrades an otherwise-GROUNDED `at_position` step to RAN when the verify CONSTANT does not match the user's PARSED coordinate goal (within tolerance) — so a model can't walk to the WRONG place and self-certify its own landing. INTRA-PACKAGE (vcli/cognitive), STRICTER-only (rule 5), no CEO gate; the NL goal co-travels with the verify in one ExecutionTrace.
- Mechanism: new pure `vcli/cognitive/coord_goal.py` (`parse_goal_coord`, `at_position_const`, `coord_goal_mismatch`); `classify_step_evidence` gains a `goal_text` kw-default (rule 6 additive). FAIL-OPEN when either side isn't a parseable coordinate. Honest by construction — the target is parsed from the USER's words, never actor-authored. Proven on real go2 sim.
- REMAINING (open): wrong-predicate-type (verify a different predicate); non-coordinate / object / room goal authenticity.

## D13 — Callable-container / reduction short-circuit closed (3rd moat review) · Accepted 2026-06-20
- 3rd review found D12's fix incomplete (rejected only LITERAL containers); a builtin container constructor (an `ast.Call`) bypassed it — `True in tuple((at_position(X,Y), True))`, `len(tuple((...,True)))==2` graded GROUNDED while the robot was nowhere near. FIX (stricter-only): new `_truth_bearing_oracle` — an oracle counts toward GROUNDED only when its VALUE reaches the operand (direct call, index/arithmetic over one, or a reduction whose arg is truth-bearing); an oracle buried in a literal/constructed container is inert. Proven RAN on real go2 sim.
- LESSON: fixing one member of a short-circuit family doesn't close it; the robust fix is a semantic "does the oracle's value reach the truth" predicate, not a node-shape blocklist.
- REMAINING (open): wrong-predicate-type on the default path; legacy-producer goal-poisoning; multi-coordinate phrasing; shape-trivial `len(get_position())==3` xfail. Moat NOT yet dry.

## D14 — Native navigation routes through the avoidance planner, not blind walk (D9 #1) · Accepted 2026-06-20
- Native gains a coordinate `navigate(x, y)` motor tool (go2/base worlds, `native_loop._NativeBaseNavigateTool`) driving `publish_goal` → FAR/local planner/lidar — the obstacle-AVOIDING route `walk_forward` lacks; system prompt steers REACH-a-place to `navigate`, `walk` only for an explicit short step.
- **Honesty invariant**: `navigate`'s cmd_vel runs on the bridge thread, GATED OUT of actor-causation (never sets `_skill_ctrl_tid`), so a `navigate` step's `at_position` verify grades **UNCAUSED → RAN** — honest "ran, can't prove the actor caused it" until causation is extended to cmd_vel. Spine BYTE-UNCHANGED; producer-side routing + prompt only.

## D15 — Wrong-predicate-type closed: a coordinate goal must be VERIFIED with a matching at_position (turn-level gate) · Accepted 2026-06-20
- Closes the wrong-predicate residual: a coordinate goal could pass verified ENTIRELY with a non-`at_position` predicate (`facing`, `len(get_position())==3`) per-step-honest but never asserting the coordinate. FIX (stricter-only, intra-package): `evidence_passed` gains a turn-gate (runs only after all-GROUNDED, can only REJECT) scoped to a go2-coordinate world — require ≥1 GROUNDED step whose verify is a bounded-tol literal `at_position` matching the commanded coord; coordinate-less go2 goals fail OPEN.
- Bonus hardening to `at_position_const`: read kwarg form `at_position(x=,y=)`; reject TOL-INFLATION (`tol`>2.0m) → fail CLOSED. Proven on real go2 sim. REMAINING (open): shape-trivial xfail; legacy goal-poisoning; object/room authenticity.

## D16 — Boolean-necessity closed: a goal's matching at_position must be a NECESSARY conjunct, not merely PRESENT (4th moat review) · Accepted 2026-06-20
- 4th review: the gates checked only that a goal-matching `at_position` was PRESENT, not NECESSARY — so a walk to the WRONG place + a verify with `at_position(goal)` buried under a disjunction/negation/threshold graded True (10 variants reproduced). FIX (stricter-only): `_necessary_at_position_consts` — only `at_position` calls reachable through `BoolOp(And)` ONLY genuinely gate the verdict; an `or`/`not`/arithmetic/IfExp sibling is excluded. Proven RAN on real go2 sim.
- LESSON: PRESENT ≠ NECESSARY; honest test is boolean necessity (reachable through AND), not lexical presence. 4th consecutive structural false-green → moat still NOT dry. This fix was written by a crashed prior round, RE-VERIFIED from scratch and LANDED (don't trust a dead round's claim). REMAINING (open): shape-trivial xfail; multi-coord parse-asymmetry; legacy goal-poisoning; object/room authenticity.
